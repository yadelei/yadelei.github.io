macOS 10.15从安装到使用全过程记录

发表于 2020-04-02 更新于 2020-09-12 分类于 计算机 ， 操作系统 ， macOS

设备与环境

MacBook Pro 13，2017款；使用时以计算机软件开发需求为主。

系统安装

拔掉MacBook上所有连接的配件，只留电源线。重启MacBook，按住Option+Command+R，连接WiFi，等待下载完成，进入macOS实用工具。先进入磁盘工具，抹掉整个磁盘。再返回到macOS实用工具，进入重新安装macOS（不需要重启电脑），然后一路继续。下载完安装文件后会自动重启安装，然后进入配置界面，按需配置。配置完成后进入桌面，即可开始正常使用。

注意：过程中偶尔动一下触控板，防止黑屏；我个人使用时，桌面经常有很多不需要同步到iCloud的文件，所以为了避免同步，在系统配置界面中，取消勾选将“文稿”和“桌面”上的文件存储在iCloud云盘中。

Homebrew

官网：https://brew.sh/

官网复制安装命令到终端执行，验证密码。

因为是通过Git克隆方式安装的，所以国内可能因为网络问题而中断（此时仍然会显示安装成功，但实际上没有），中断了可以重新运行命令，直到真正安装成功为止。

可以通过命令brew doctor来检验是否安装成功。

Brew Cask默认是未启用的，运行运行一次brew cask命令来启用，Brew Cask也是需要通过Git克隆来安装，所以也可能面临网络问题。

鉴于Homebrew及Cask的数据越来越大，所以在国内使用时网络是个大问题，不挂梯子经常就是小水管，时间长不说，关键几乎每次都一定会在中途断掉，所以基本上现在我安装Homebrew时一定要上挂梯子。一般在安装Homebrew时（即克隆GitHub项目时）速度不快（不能持续在100KB/s以上），那就想办法用别的网络渠道吧，因为很可能等不到完成就中断了。

网易有道词典

网易有道词典在使用取词/划词功能时，需要开启系统隐私中的“屏幕录制”权限，这就涉及到一些信息安全的问题，所以，尽量不要在屏幕上有敏感信息的时候使用有道词典。

目前来说，有道词典在划词方面已经有所改进，不再出现划词、取词是同一个功能的情况，但是体验仍然比Windows版和Android版差得远。

简言之，网易有道词典Mac版只适合作为临时的、需求简单时的查词工具使用，不适合作为大量英文文本阅读的时候使用，这种情况下应该使用Windows。

大量英文阅读方案

鉴于Mac版的有道词典比较弱，所以在Mac上，在需要进行大量英文阅读时，最佳方案是使用Windows虚拟机：

  • 虚拟机软件：VMware Fusion
  • Windows版本：Windows 10
  • 分辨率：与Mac相同的渲染分辨率和缩放率（一般是2880×1800分辨率，200%缩放率）
  • 有道词典查词结果窗口可以固定在一旁

Ubuntu 18.04常用软件HiDPI高分屏设置

发表于 2018-06-28 更新于 2020-07-01 分类于 计算机 ， 操作系统 ， Linux

我的电脑是小米笔记本 Air 13.3，分辨率1920x1080（1080P）。这个分辨率在这个尺寸下有点尴尬，日常使用必须将GUI放大（否则界面元素太小，看着累），又不能像苹果Retina屏一样整数倍放大（否则可用界面空间太小，不方便操作）。

我这台电脑上安装Windows系统时，Windows会自动设置一个缩放率150%，但是安装Linux系统如Ubuntu，就必须手动去设置缩放了。查阅资料后，最终把Ubuntu 18.04下的缩放问题基本解决了。不得不讲，Linux相比Windows对高分屏的支持还是要好不少的，只不过Linux在GUI这块儿派系比较多，所以设置起来可能要考虑多个方面的东西。

环境和参数

操作系统 桌面环境 屏幕分辨率 屏幕尺寸 缩放因子
Ubuntu Desktop 18.04 LTS Gnome 3.28.1 1920x1080 13.3英寸 1.33（缩放后相当于1440x810）

Gnome全局设置

Ubuntu 18.04使用的是Gnome桌面环境，所以这里的设置方法对其他安装了Gnome桌面环境的发行版也适用。

可行方案

Gnome下真正有效的、通用的方案是，使用文本缩放因子（Text Scaling Factor）。放大后的文字都很清晰，其他界面元素也基本都会跟着文字一起放大。

设置文本缩放因子的方法主要有以下几种：

  • 在设置 > 通用辅助功能下开启大号文本，默认是将文本缩放至**125%**，不可自定义缩放因子（缩放倍数）；
    同时，也可以在这里将鼠标的光标大小设置为中等，以匹配缩放后的观感；

  • 在Gnome Tweaks软件的字体中手动设置缩放比例的数值，可以自定义缩放因子，甚至缩小文本都可以；

Firefox/Thunderbird设置

在进行Gnome全局设置后，很多程序就已经能够正常缩放了，但是Mozilla家常用的两个数码宝贝，火狐（Firefox）和雷鸟（Thunderbird）还点问题，具体表现为：程序界面字体正常放大，但是界面比较紧凑，而内容显示区域（网页和邮件内容）并没有放大。

此时需要对Firefox和Thunderbird进行单独设置，设置的方法都是在配置编辑器（about:config）中修改layout.css.devPixelsPerPx项的值。
